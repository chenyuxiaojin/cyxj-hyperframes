# Optional Screen Recording Replacement List

Terminal, Claude Code UI, statusline, hooks, worktree, and panel scenes are generated motion graphics. Do not replace them with reference footage.

| Time | Segment | Why real recording may help | Suggested recording | Current generated fallback |
|---|---|---|---|---|
| 05:35 | Screenshot bug handoff | A real error page or broken UI can make the visual-input point more concrete. | One cropped browser/app error screenshot dragged into Claude Code. | Generated screenshot dropzone and vision state panel. |
| 07:29 | Phone remote control | QR scanning and mobile browser control are more believable with actual device footage. | Short phone-over-desk shot scanning a remote-session QR and controlling the session. | Generated QR card and remote-session panel. |
| 08:00 | External inspiration/source page | A real webpage can clarify when to paste API docs or examples instead of installing MCP. | Browser capture of an API docs page, cropped to avoid private data. | Generated API-vs-MCP compare cards. |
| 09:10 | Skill Creator structure | A true project tree can make repeatable skill packaging easier to recognize. | Finder or editor tree showing `SKILL.md`, `scripts/`, and `examples/`. | Generated Skill Creator folder assembly. |
| 05:04 | System completion notification | If the generated toast feels too abstract, the real macOS notification can add credibility. | Clean macOS notification after a local hook finishes. | Generated hooks settings panel and notification toast. |
