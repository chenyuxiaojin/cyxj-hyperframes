 操作准则（OFFICIAL_DOCS_VALUE_INDEX 7 条 ⭐⭐⭐ 洞察 + CLAUDE.md 硬约束）

 这 11 条都是已验证的硬规则，违反就 lint 报错或视觉烂。

 1. 相对时序串联 19 招（concepts/data-attributes.md ⭐⭐⭐）

 <div data-composition-id="tip-01-statusline" data-start="hook + 0" data-duration="32"></div>
 <div data-composition-id="tip-02-commit" data-start="tip-01-statusline + 0" data-duration="17"></div>

 改第 7 招时长，后面所有自动平移。chain 不超过 3-4 层（用户体验 + lint 限制）。

 1. HTML 声明结构 / Script 只做动画（reference/html-schema.md ⭐⭐⭐）

 HTML = "什么播什么时候播"，GSAP script = "动画效果"。两层职责严格分离，不混。

 1. 每个 composition 必须 register timeline + pad 到 ≥ data-duration（HF 硬规则）

 const tl = gsap.timeline();
 // ... 动画 ...
 tl.set({}, {}, 32);  // 撑满 32 秒，避免 timeline 提前结束
 window.__timelines["tip-01-statusline"] = tl;

 key 必须等于 data-composition-id。

 1. GSAP querySelector 硬编码字符串（CLAUDE.md 已记录硬约束）

 // ❌ 报错 template_literal_selector
 gsap.to(`${root} .b1-card`, ...)
 // ✅
 gsap.to('[data-composition-id="tip-01-statusline"] .b1-card', ...)

 1. 不要手动 animate <video> 的尺寸（guides/gsap-animation.md）

 包 wrapper div 再 animate，否则 seek-and-capture 抓不到正确帧。

 1. 禁用 Math.random() / Date.now()（concepts/determinism.md）

 不带种子的随机数会让每帧不一致，渲染确定性破坏。

 1. backdrop-filter blur 叠层 ≤ 2-3 层 / region（guides/performance.md）

 每 region 超 3 层 blur，单帧 200ms+。米色 hero scene 用透明卡片要克制。

 1. 图片源 ≤ 2× canvas 分辨率（guides/common-mistakes.md）

 1920×1080 工程不放 7000×5000 图，140MB 解码内存浪费。

 1. 复制 beat html 时全局换 beat id（CLAUDE.md 已记录）

 CSS class 前缀 + GSAP selector 都要换。sed -i '' 's/old-id/new-id/g' file.html。

 1. 转场三件套（catalog/blocks/whip-pan + flash-through-white + cinematic-zoom）

- whip-pan —— 招与招之间默认转场（不硬切）
- flash-through-white —— 章节大切（如开场→第一招、第十招→第十一招这种节点）
- cinematic-zoom —— 重点强调（最多 2-3 次，避免审美疲劳）

 1. 颗粒 + 暖暗角全程挂在 index.html 顶层（DNA 三件套，grain-overlay）

 不在每个 beat 里画 grain。Index.html 顶层挂一次：

- grain-overlay 透明度 0.10-0.12，mix-blend-mode overlay
- 暖色 vignette（不是黑色）：radial-gradient(ellipse 75% 75%, transparent 30%, rgba(43, 38, 34, 0.15) 100%)

- 透视网格深色细线 → 透视网格 rgba(43,38,34,0.06) 0.06 透明度（米色背景上几乎不可见但有"骨架"）
